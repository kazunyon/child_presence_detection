# まもるバス

**まもるバス**は、保育園・幼稚園の送迎バスで、子どもの置き去りを防ぐための安全確認PWAです。

高価な専用機器だけに頼らず、園にあるスマートフォンとQRコードを活用し、職員が毎日行う確認を

**乗車 → 降車 → 車内の目視確認 → 最後尾確認 → 別の職員による確認 → 記録保存**

の順番で、迷わず進められる仕組みを目指しています。

画面は、ITに詳しくない職員でも使いやすいように、大きなボタンと分かりやすい日本語を中心にしています。AIは職員の確認を補助する役割とし、AIの判定だけで安全確認を完了させない設計方針です。

> [!IMPORTANT]
> 現在は開発途中の試作版です。公開ページは画面と基本操作を確認するためのデモであり、実際の送迎業務にはまだ使用できません。

## 公開デモ

[まもるバスを開く](https://kazunyon.github.io/child_presence_detection/)

デモ用のログイン情報です。

- 職員ID：`1`
- PIN：`1234`
- 最後尾QRの代替文字列：`bus-tail-2`
- 第三者確認用PIN：`5678`

GitHub Pagesで公開しているのはReactの画面部分です。FastAPIとSQLiteはGitHub Pages上では動作しないため、公開デモで操作した内容は正式な記録として保存されません。

## 現在できること

### 公開デモでできること

- スマートフォン向けの「ホーム・園児・記録・設定」画面を表示
- デモ用の職員IDとPINによるログイン
- カメラを使ったQRコード読み取り
  - ブラウザがQR自動読取に対応していない場合は、QRの文字列を手入力可能
- 降車確認から、最後尾QR確認、第三者確認までの一連の操作体験
- 最後尾確認時のGPS取得
  - 位置情報を許可した端末のみ
- 園児・職員・車両・便を登録する設定画面の操作体験
- PWAとしての基本設定
  - 対応端末・ブラウザでは、ホーム画面への追加またはインストールが可能
- GitHub ActionsによるGitHub Pagesへの自動公開

### FastAPIを起動した開発環境でできること

- SQLiteへの基本データ保存
- 職員IDとPINの照合
- 園児・職員・車両・便の登録・一覧取得API
- 園児QRによる降車記録の保存
- 最後尾QR確認の日時・担当者・GPS位置の保存
- 環境変数 `DATABASE_URL` を利用できるデータベース接続構成

> フロント画面には固定のデモデータも含まれています。APIに保存したデータが、すべての画面へ自動反映される段階ではありません。

## 現在できないこと

### 最重要：公開PWAは「操作デモ」であり、実運用には使えません

GitHub Pagesで公開しているまもるバスは、便の開始、乗車・降車、未降車ブロック、最後尾確認、第三者承認という**操作の流れを確認するためのデモ**です。

公開PWAでは、バックエンドへ接続できない場合に画面上だけで動くデモモードへ切り替わります。このとき、画面に表示される人数・園児状態・完了表示は正式記録ではなく、再読み込みすると消えます。

**園児の安否確認、実際の送迎業務、監査記録、法令上必要な安全確認には使用しないでください。** 必ず職員による車内の目視確認と、園の緊急対応手順を優先してください。

### 1. 実データの保存と共有

現在、便ごとの乗車・降車APIとPWA画面の接続処理は実装されています。しかし、公開先のGitHub PagesにはFastAPIとデータベースがありません。

- 公開PWAから操作した便・乗車・降車・車内確認は、通常は正式なデータベースに保存されません。
- 他の職員のスマートフォンで同じ便を開いても、リアルタイムに人数や園児状態は共有されません。
- ページを閉じる・再読み込みする・端末を変えると、デモモードの記録は失われます。
- 実保存するには、FastAPIをHTTPSでクラウド公開し、PWAの `VITE_API_BASE_URL` に接続先を設定する必要があります。

### 2. 乗車・降車の対象範囲

便単位で「乗車数・降車数・未降車数」を管理し、未降車がいる間は後続の安全確認を止める画面は実装済みです。ただし、次は未実装です。

- 園児ごとの利用予定、欠席、当日変更、途中乗車・途中降車の管理
- 保護者への確実な引渡し確認
- QRコードの発行、印刷、紛失時の再発行、失効、重複防止の運用画面
- 複数の車両・複数便を同時に運行する際の配車・担当割当
- 乗降記録の修正、取消し、修正理由の監査ログ

### 3. 車内確認と第三者承認

最後尾QRを読む画面と第三者承認の操作フローはありますが、実運用に必要な証跡の強度には達していません。

- 30秒の車内確認動画を撮影、アップロード、再生、保管する機能はありません。
- AIによる画像・動画の補助判定はありません。
- 第三者承認は現時点ではデモ用PINです。
  - 運転担当者と第三者確認者が本当に別人かを、個別アカウント・セッション・端末認証で保証していません。
  - 承認の日時、担当者、根拠を改ざん検知可能な形で保存していません。
- 最後尾QR・GPSの結果も、公開デモでは正式な証跡として保管されません。

### 4. 未確認アラームと関係者通知

画面アラームと通知キューの土台はありますが、実際に関係者へ確実に届ける通知機能は未接続です。

- 「未確認から15分経過」などをサーバー側で常時監視し、自動発報する機能はありません。
- 管理者へのPush通知、メール、SMS、電話連携はありません。
- 保護者への降車完了通知は、LINE・メール等へ実送信されません。
- 送信成功・未達・既読・再通知を追跡する仕組みはありません。
- 通知先や送信内容を園ごとに管理する画面はありません。

### 5. 記録、監査、障害対応

- 「記録」画面は見本であり、実際の運行履歴を検索・閲覧する画面ではありません。
- 証跡をPDF/CSVで出力すること、長期保管すること、監査に提出することは未実装です。
- 通信が切れたときに端末へ安全に保留し、復旧後に重複なく送信するオフライン対応はありません。
- バックアップ、障害復旧、誤登録時の復旧手順、データ削除・保存期間管理はありません。

### 6. 本番用の認証・個人情報保護

- 個別職員アカウント、PIN変更、ログイン有効期限、多要素認証、役割ごとの権限設定は未実装です。
- 園児・保護者情報、GPS、動画等の暗号化とアクセス制御は未実装です。
- PostgreSQLへの移行・性能試験・負荷試験は未実施です。
- 実際の園での運用試験、誤操作・通信不良・緊急時の訓練、安全管理責任者等による確認は未実施です。

> **次の優先作業**：FastAPIとデータベースをHTTPSで公開し、公開PWAから実保存できるようにします。その後、実データの記録一覧、個別職員認証、通知サービス連携を順に実装します。
## 目指す運用の流れ

1. 園児が乗車したら、園児QRを読み取ります。
2. 到着後、園児が降車するたびにQRを読み取ります。
3. 乗車人数と降車人数が一致したことを確認します。
4. 職員が座席、足元、座席の下、荷物の陰を目視確認します。
5. 車両の最後尾に設置したQRを読み取ります。
6. 車内を30秒撮影し、AIが確認を補助します。
7. 運転担当者とは別の職員が、車内と記録を再確認します。
8. 第三者承認後、日時・GPS・担当者・確認記録を保存します。
9. 必要に応じて、保護者へ降車完了を通知します。

> この流れは完成時の目標です。現時点では、すべての工程が実装済みではありません。

## 安全上の考え方

まもるバスは、職員による安全確認を支援するシステムです。

- アプリの表示だけで「安全」と判断しない
- 必ず職員が車内の最後尾まで移動して目視確認する
- 運転担当者とは別の職員が再確認する
- 人数が一致しない場合は完了扱いにしない
- 園児の所在が分からない場合は、園の緊急対応手順に従う

スマートフォン、QRコード、動画、AIのいずれか一つに頼るのではなく、複数の確認を重ねて見落としを減らすことを目的としています。

## 開発環境

### フロントエンド

- React
- TypeScript
- Vite
- Tailwind CSS
- vite-plugin-pwa

```powershell
npm install
npm run dev
```

### バックエンド

- FastAPI
- Python
- SQLAlchemy
- SQLite（開発用）

```powershell
cd backend
python -m pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

フロントエンドは、環境変数 `VITE_API_BASE_URL` でFastAPIのURLを指定できます。未指定時は `http://127.0.0.1:8000` を使用します。

## GitHub Pagesへの公開

`main` ブランチへ変更を反映すると、GitHub Actionsがフロントエンドをビルドし、GitHub Pagesへ公開します。

初回だけ、GitHubリポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定してください。
