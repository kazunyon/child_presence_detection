# まもるバス

**まもるバス**は、保育園・幼稚園の送迎バスで、子どもの置き去りを防ぐための安全確認PWAです。

高価な専用機器だけに頼らず、園にあるスマートフォンとQRコードを活用し、職員が毎日行う確認を

**乗車 → 降車 → 車内の目視確認 → 最後尾確認 → 別の職員による確認 → 記録保存**

の順番で、迷わず進められる仕組みを目指しています。

画面は、ITに詳しくない職員でも使いやすいように、大きなボタンと分かりやすい日本語を中心にしています。AIは職員の確認を補助する役割とし、AIの判定だけで安全確認を完了させない設計方針です。

> [!IMPORTANT]
> 現在は開発途中の試作版です。公開ページは画面と基本操作を確認するためのデモであり、実際の送迎業務にはまだ使用できません。

## 公開デモ

[まもるバスを開く](https://kazunyon.github.io/child_presence_detection/)

デモ用のログイン情報です。

- 職員ID：`1`
- PIN：`1234`
- 最後尾QRの代替文字列：`bus-tail-2`

GitHub Pagesで公開しているのはReactの画面部分です。FastAPIとSQLiteはGitHub Pages上では動作しないため、公開デモで操作した内容は正式な記録として保存されません。

## 現在できること

### 認証・園ごとのデータ分離

- APIログインでJWTを発行し、以降のAPI操作は認証トークンを必須にする
- 職員には `admin`、`operator`、`verifier` のロールを設定し、管理操作・通知送信・動画判定を制限する
- 園（organization）IDを園児、職員、車両、便、通知、監査ログ、動画証跡の全レコードに持たせ、別園のデータをAPIから取得・操作できないようにする
- PINはPBKDF2-SHA256でハッシュ化して保存する

### 運行記録・監査・オフライン

- 便の作成、乗車・降車、最後尾QR（GPS許可時は位置情報も）、第三者確認、便完了をAPIへ記録する
- 未降車の園児がいる便、または最後尾確認のない便は完了できない
- 運行一覧を期間指定で取得し、管理者は認証・作成・乗降・完了・通知・動画操作の監査ログを検索できる
- 通信不能時の乗降記録を端末内に保留し、復帰後に `client_event_id` による重複防止付きで同期する

### 通知・動画・AI連携の土台

- 通知をキューへ登録し、`NOTIFICATION_WEBHOOK_URL` に接続したWebhookへ送信できる
- 便ごとの動画をアップロードし、動画証跡として保存できる
- AI判定リクエストの状態を記録する（AIプロバイダー未接続時は必ず人による目視確認が必要と表示する）

## 現在できないこと・導入前の必須対応

### 実際の送迎業務には、まだ使用できません

本アプリは開発途中です。園児の安否確認、事故防止の最終確認、法令上必要な記録には使用しないでください。必ず職員による目視確認と園の緊急対応手順を優先してください。

- 第三者確認は画面上のPINであり、運転者本人との厳密な分離や承認者の本人性確認は未完成です。
- Webhook以外のメール、SMS、Push、LINE等の通知事業者接続、再送、到達確認は未実装です。
- AI動画判定はプロバイダー未接続で、実際の検出・判定は行いません。AIの結果だけで便を完了させる設計にはしていません。
- 動画はローカルファイル保存です。本番では暗号化されたオブジェクトストレージ、保存期限、削除手順、アクセス記録を整備してください。
- パスワード再設定、多要素認証、トークン失効、PIN変更、管理画面、PDF/CSV出力、バックアップ／障害復旧は未実装です。
- SQLiteから既存のデータベースを移行するためのAlembicマイグレーション、負荷試験、脆弱性診断、実園での運用評価は未実施です。
## 目指す運用の流れ

1. 園児が乗車したら、園児QRを読み取ります。
2. 到着後、園児が降車するたびにQRを読み取ります。
3. 乗車人数と降車人数が一致したことを確認します。
4. 職員が座席、足元、座席の下、荷物の陰を目視確認します。
5. 車両の最後尾に設置したQRを読み取ります。
6. 車内を30秒撮影し、AIが確認を補助します。
7. 運転担当者とは別の職員が、車内と記録を再確認します。
8. 第三者承認後、日時・GPS・担当者・確認記録を保存します。
9. 必要に応じて、保護者へ降車完了を通知します。

> この流れは完成時の目標です。現時点では、すべての工程が実装済みではありません。

## 安全上の考え方

まもるバスは、職員による安全確認を支援するシステムです。

- アプリの表示だけで「安全」と判断しない
- 必ず職員が車内の最後尾まで移動して目視確認する
- 運転担当者とは別の職員が再確認する
- 人数が一致しない場合は完了扱いにしない
- 園児の所在が分からない場合は、園の緊急対応手順に従う

スマートフォン、QRコード、動画、AIのいずれか一つに頼るのではなく、複数の確認を重ねて見落としを減らすことを目的としています。

### フロントエンド

- React
- TypeScript
- Vite
- Tailwind CSS
- vite-plugin-pwa

```powershell
npm install
npm run dev
```

### バックエンド

- FastAPI
- Python
- SQLAlchemy
- SQLite（開発用）

```powershell
cd backend
python -m pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

フロントエンドは、環境変数 `VITE_API_BASE_URL` でFastAPIのURLを指定できます。未指定時は `http://127.0.0.1:8000` を使用します。

## LINE通知の設定

LINE Messaging APIのPush通知を実装しています。LINE公式アカウントを友だち追加した保護者・職員のLINEユーザーIDを、署名検証済みWebhookから園ごとに登録し、そのIDを宛先として通知を送ります。

1. LINE DevelopersでLINE公式アカウントに紐付くMessaging APIチャネルを作成します。
2. Renderへデプロイ後、Webhook URLを `https://<APIのURL>/api/integrations/line/webhook` に設定してWebhookを有効化します。
3. Renderの環境変数へ `LINE_CHANNEL_ACCESS_TOKEN`、`LINE_CHANNEL_SECRET`、`LINE_ORGANIZATION_ID` を登録します。トークンとSecretをGitやブラウザへ保存してはいけません。
4. 保護者または職員が公式アカウントを友だち追加すると、WebhookでLINEユーザーIDが登録されます。管理者は `GET /api/integrations/line/contacts` で登録済みの宛先を確認できます。
5. 通知は `channel: "line"`、宛先に登録済みのLINEユーザーID（`U`で始まる値）を指定してキューへ入れ、管理者が `POST /api/notifications/{id}/dispatch` を実行すると送信されます。

LINEの設定が未完了の場合、通知は送信済みにならず `failed` として記録されます。テスト時は実在する保護者へ送らず、専用のテスト用LINEアカウントを使ってください。
## GitHub Pagesへの公開

`main` ブランチへ変更を反映すると、GitHub Actionsがフロントエンドをビルドし、GitHub Pagesへ公開します。

初回だけ、GitHubリポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定してください。
