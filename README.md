# まもるバス

**まもるバス**は、保育園・幼稚園の送迎バスで、子どもの置き去りを防ぐための安全確認PWAです。

高価な専用機器だけに頼らず、園にあるスマートフォンとQRコードを活用し、職員が毎日行う確認を

**乗車 → 降車 → 車内の目視確認 → 最後尾確認 → 別の職員による確認 → 記録保存**

の順番で、迷わず進められる仕組みを目指しています。

画面は、ITに詳しくない職員でも使いやすいように、大きなボタンと分かりやすい日本語を中心にしています。AIは職員の確認を補助する役割とし、AIの判定だけで安全確認を完了させない設計方針です。

> [!IMPORTANT]
> 現在は開発途中の試作版です。公開ページは画面と基本操作を確認するためのデモであり、実際の送迎業務にはまだ使用できません。

## 公開デモ

[まもるバスを開く](https://kazunyon.github.io/child_presence_detection/)

デモ用のログイン情報です。

- 職員ID：`1`
- PIN：`1234`
- 最後尾QRの代替文字列：`bus-tail-2`
- 第三者確認用PIN：`5678`

GitHub Pagesで公開しているのはReactの画面部分です。FastAPIとSQLiteはGitHub Pages上では動作しないため、公開デモで操作した内容は正式な記録として保存されません。

## 現在できること

### 公開デモでできること

- スマートフォン向けの「ホーム・園児・記録・設定」画面を表示
- デモ用の職員IDとPINによるログイン
- カメラを使ったQRコード読み取り
  - ブラウザがQR自動読取に対応していない場合は、QRの文字列を手入力可能
- 降車確認から、最後尾QR確認、第三者確認までの一連の操作体験
- 最後尾確認時のGPS取得
  - 位置情報を許可した端末のみ
- 園児・職員・車両・便を登録する設定画面の操作体験
- PWAとしての基本設定
  - 対応端末・ブラウザでは、ホーム画面への追加またはインストールが可能
- GitHub ActionsによるGitHub Pagesへの自動公開

### FastAPIを起動した開発環境でできること

- SQLiteへの基本データ保存
- 職員IDとPINの照合
- 園児・職員・車両・便の登録・一覧取得API
- 園児QRによる降車記録の保存
- 最後尾QR確認の日時・担当者・GPS位置の保存
- 環境変数 `DATABASE_URL` を利用できるデータベース接続構成

> フロント画面には固定のデモデータも含まれています。APIに保存したデータが、すべての画面へ自動反映される段階ではありません。

## 現在できないこと

### まずご理解ください：公開デモは実運用には使えません

公開しているGitHub Pages版は、画面と操作の流れを確認するためのデモです。入力した内容が、園の正式な安全記録として確実に保存・共有・通知される状態ではありません。**実際の送迎業務、園児の安否確認、法令上必要な安全確認には使用しないでください。**

### 1. 乗車・降車の完全な照合

- **画面操作と「便ごとの記録」がまだ完全には接続されていません。**
  - バックエンドには便ごとの乗車・降車・未降車人数を管理するAPIがありますが、公開PWA画面はまだこのAPIを使っていません。
- そのため、画面に表示される人数や園児状態は、実際のQR読み取り結果と一致する保証がありません。
- 「乗車人数と降車人数が一致しない場合に、車内確認や完了操作を必ず止める」制御は、画面上では未完成です。

### 2. QR読み取り・データ保存

- カメラでQRを読み取る画面はありますが、端末やブラウザによっては自動読取に対応していません。
- GitHub PagesだけではFastAPIやSQLiteを動かせないため、公開デモで操作した乗降・車内確認は正式なデータベースへ保存されません。
- 園児QRの発行、印刷、紛失時の再発行、失効管理は未実装です。
- 設定画面で追加した園児・職員・車両・便は、公開デモではページを再読み込みすると元に戻ります。

### 3. 車内確認・第三者承認

- 30秒の車内確認動画を撮影し、アップロード・再生・長期保管する機能は未実装です。
- AIで画像や動画を判定し、車内に子どもが残っていないかを補助確認する機能は未実装です。
- 第三者確認は現在デモ用PINによる画面上の操作です。
  - 運転担当者と別人であることを、個別アカウントやログイン履歴で厳密に保証する仕組みはありません。
  - 第三者承認の結果を監査可能な形で保存する機能も未完成です。

### 4. アラーム・通知

- 未確認を検知した際の画面アラームと通知キューはありますが、**15分経過などを自動監視して確実に発報する仕組みは未実装**です。
- 管理者へのプッシュ通知、メール、SMS、電話連携は未接続です。
- 保護者への「降車完了」通知も、現時点では実際にLINE・メール等へ送信されません。
- 通知を誰へ送ったか、届いたか、確認されたかを追跡する機能はありません。

### 5. 記録・監査・複数端末運用

- 「記録」画面に表示される内容は見本であり、実データベースの記録一覧ではありません。
- 証跡をPDF出力すること、長期間保管すること、検索・監査することは未実装です。
- 複数のスマートフォンで同じ便の状態をリアルタイム共有する機能はありません。
- 通信が切れた場合に端末へ安全に記録を保留し、復旧後に重複なく再送するオフライン対応は未実装です。

### 6. 本番運用に必要な安全性・セキュリティ

- 個別アカウント、PIN変更、ログイン有効期限、権限ごとの操作制限、監査ログは未実装です。
- 個人情報の暗号化、保存期間の設定と削除、バックアップ、障害復旧は未実装です。
- PostgreSQLへの実接続・移行試験は未実施です。
- 実際の園での操作試験、通信不良時の試験、誤操作時の試験、専門家による安全性確認はまだ行っていません。

> **次に優先すべきこと**：FastAPIをクラウドへ公開し、PWA画面を便ごとの乗降記録APIへ接続することです。これにより「誰が・どの便で・いつ乗車／降車したか」を実データとして保存し、未降車なら完了できない仕組みを画面にも反映できます。
## 目指す運用の流れ

1. 園児が乗車したら、園児QRを読み取ります。
2. 到着後、園児が降車するたびにQRを読み取ります。
3. 乗車人数と降車人数が一致したことを確認します。
4. 職員が座席、足元、座席の下、荷物の陰を目視確認します。
5. 車両の最後尾に設置したQRを読み取ります。
6. 車内を30秒撮影し、AIが確認を補助します。
7. 運転担当者とは別の職員が、車内と記録を再確認します。
8. 第三者承認後、日時・GPS・担当者・確認記録を保存します。
9. 必要に応じて、保護者へ降車完了を通知します。

> この流れは完成時の目標です。現時点では、すべての工程が実装済みではありません。

## 安全上の考え方

まもるバスは、職員による安全確認を支援するシステムです。

- アプリの表示だけで「安全」と判断しない
- 必ず職員が車内の最後尾まで移動して目視確認する
- 運転担当者とは別の職員が再確認する
- 人数が一致しない場合は完了扱いにしない
- 園児の所在が分からない場合は、園の緊急対応手順に従う

スマートフォン、QRコード、動画、AIのいずれか一つに頼るのではなく、複数の確認を重ねて見落としを減らすことを目的としています。

## 開発環境

### フロントエンド

- React
- TypeScript
- Vite
- Tailwind CSS
- vite-plugin-pwa

```powershell
npm install
npm run dev
```

### バックエンド

- FastAPI
- Python
- SQLAlchemy
- SQLite（開発用）

```powershell
cd backend
python -m pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

フロントエンドは、環境変数 `VITE_API_BASE_URL` でFastAPIのURLを指定できます。未指定時は `http://127.0.0.1:8000` を使用します。

## GitHub Pagesへの公開

`main` ブランチへ変更を反映すると、GitHub Actionsがフロントエンドをビルドし、GitHub Pagesへ公開します。

初回だけ、GitHubリポジトリの **Settings → Pages → Build and deployment → Source** を **GitHub Actions** に設定してください。
