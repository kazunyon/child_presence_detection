# まもるバス

保育園・幼稚園向けの、送迎バス置き去り防止・安全確認PWAです。既存のスマートフォンとQRシールで、**乗車 → 降車 → 車内確認 → 第三者確認 → 記録**を一続きで管理します。

## できること

- 園児ごとのQRコードによる乗車・降車記録
- 最後尾QRと30秒の車内確認動画による、車内の安全確認
- AI画像チェックと、別の職員による第三者承認
- 未確認時のアラームと管理者への通知
- 日時・GPS位置・確認者を含む証跡の保存
- 保護者への「降車完了」通知（通知サービス連携時）

> **大切なお願い**：本システムは安全確認を支援する道具です。画面の記録だけに頼らず、必ず職員が実際に車内を目視確認してください。

## ご利用前の準備

1. 園で「運転担当」「降車確認担当」「第三者確認担当」「管理者」を決めます。同一人物だけで完結させないことが重要です。
2. 園児ごとのQRコードと、各車両の最後尾確認用QRコードを発行・印刷し、ラミネートします。
3. 園のスマートフォンで [まもるバス](https://kazunyon.github.io/child_presence_detection) を開きます。
4. ブラウザのメニューから「ホーム画面に追加」または「アプリをインストール」を選びます。以後はアプリアイコンから開けます。
5. 初回起動時に、カメラ・位置情報・通知をすべて「許可」してください。許可しない場合、QR読み取り・GPS証跡・未確認通知が利用できません。

## インストールガイド（職員向け）

### iPhone / iPad（Safari）

1. Safariで [まもるバス](https://kazunyon.github.io/child_presence_detection) を開きます。
2. 画面下部の共有ボタン（□に↑）を押します。
3. **「ホーム画面に追加」** を選び、右上の「追加」を押します。
4. ホーム画面の「まもるバス」を押して起動します。

### Android（Chrome）

1. Chromeで [まもるバス](https://kazunyon.github.io/child_presence_detection) を開きます。
2. 右上のメニュー（︙）を押します。
3. **「アプリをインストール」** または **「ホーム画面に追加」** を選びます。
4. 確認画面で「インストール」を押し、ホーム画面のアイコンから起動します。

## 使い方（毎日の運行マニュアル）

### 1. 運行開始前

1. 「まもるバス」を開き、日付・便・車両・担当者が正しいことを確認します。
2. 端末の充電、通信状態、カメラとGPSの許可状態を確認します。
3. QRシールが読める位置にあることを確認します。

### 2. 乗車時：園児QRを読み取る

1. ホームの **「QRを読み取る」** を押します。
2. 園児カードのQRをカメラ枠に入れます。
3. 園児名が表示されたら、本人と名前を声に出して照合します。
4. 「乗車済み」になったことを確認します。

### 3. 降車時：園児QRを読み取る

1. 降車する園児のQRを読み取ります。
2. 園児が保護者、または受け入れ担当者へ確実に引き渡されたことを目視確認します。
3. 画面で「降車済み」になったことを確認します。
4. 保護者通知を有効にしている場合は、降車完了通知を送信します。

### 4. 全員の降車後：車内を確認する

1. 一覧で、乗車した園児全員が「降車済み」になっていることを確認します。**1人でも未確認なら、車内確認へ進めません。**
2. 運転担当者または確認担当者が、座席の間・足元・荷物の陰まで後方へ歩いて目視確認します。
3. 最後尾のQRコードを読み取り、30秒の車内確認動画を撮影します。
4. AI画像チェックの結果を確認します。「要確認」と表示された場合は、再度車内を目視確認してから管理者へ連絡します。

### 5. 第三者確認（ダブルチェック）

1. 運転担当者とは別の職員が、園児一覧・最後尾QR・撮影記録を確認します。
2. 実際の車内をもう一度確認します。
3. 問題がなければ「第三者承認」を実行します。
4. ホームに **「本日の安全確認が完了しました」** と表示されたことを確認します。

### 6. 未確認アラームが出たとき

1. アラームを止める前に、園児一覧の「未確認」を確認します。
2. 直ちに車両へ向かい、乗車名簿・保護者・担任と照合します。
3. 園児の所在確認が取れるまで、安全確認を完了扱いにしないでください。
4. 所在不明の可能性が少しでもある場合は、園の緊急対応手順に従い、管理者・警察・保護者へ連絡します。

## 記録の確認

画面下部の **「記録」** から、便ごとに以下を確認できます。

- 乗車・降車の時刻と確認者
- GPS位置情報
- 車内確認の動画、AI判定結果
- 第三者承認の時刻と担当者

記録は事故防止と監査のための重要な証跡です。園の情報管理規程に従って保管してください。

## 開発・管理者向けセットアップ

### フロントエンド

```powershell
npm install
npm run dev
```

### バックエンド

別のターミナルで実行します。

```powershell
cd backend
python -m pip install -r requirements.txt
uvicorn main:app --reload --port 8000
```

開発時のデータベースはSQLiteです。環境変数 `DATABASE_URL` にPostgreSQL接続URLを指定すると、本番用データベースへ切り替えられます。

## GitHub Pagesへの公開

`main` ブランチへプッシュすると、GitHub Actionsが自動でPWAをビルドしてGitHub Pagesへ公開します。初回のみ、リポジトリの **Settings → Pages → Build and deployment** でソースを **GitHub Actions** に設定してください。
